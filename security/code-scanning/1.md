## What CodeQL reported

The fast-forward merge step references a first-party action by a floating major tag rather than a full commit SHA:

```yaml
- name: Fast-forward merge
  uses: bitwise-media-group/ff-merge@v1
```

A tag can be re-pointed at a different commit, so pinning to `@v1` means the bytes that run can change without a
corresponding change in this repository. CodeQL recommends pinning to a full commit SHA (as every third-party action in
this repository already is, e.g. `actions/checkout@df4cb1c...`).

## Status

Not yet triaged. This is unrelated to the untrusted-checkout findings (#2, #3) — it is a supply-chain pinning question
about a first-party `bitwise-media-group` action. Disposition to decide:

- **Remediate** — pin `ff-merge` to a full commit SHA with a `# v1` comment, matching the repository's convention for
  third-party actions, and let Dependabot bump it.
- **Accept / dismiss** — if floating the major tag of a first-party action under the same org's control is an
  intentional release-coupling choice, document that reasoning and dismiss.
