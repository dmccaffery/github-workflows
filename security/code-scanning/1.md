## What CodeQL reported

The fast-forward merge step references a first-party action by a floating major tag rather than a full commit SHA:

```yaml
- name: Fast-forward merge
  uses: bitwise-media-group/ff-merge@v1
```

A tag can be re-pointed at a different commit, so pinning to `@v1` means the bytes that run can change without a
corresponding change in this repository. CodeQL recommends pinning to a full commit SHA (as every third-party action in
this repository already is, e.g. `actions/checkout@df4cb1c...`).

## Resolution

Pinned to the full commit SHA that `v1` currently resolves to, with a version comment, matching the repository's
convention for third-party actions:

```yaml
- name: Fast-forward merge
  uses: bitwise-media-group/ff-merge@182b1c83d954664a09eca1aeee252873504a329a # v1.0.0
```

`182b1c8` is the commit behind both `v1` and `v1.0.0` at the time of pinning. Dependabot reads the `# v1.0.0` comment
and will open a PR to bump the SHA when a new `ff-merge` release is published, so the pin stays current without floating
on a re-pointable tag.
